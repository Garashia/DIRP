# dirp DSL 技術仕様（日本語）

**Version:** 1.0  
**拡張子:** `.dirp`  
**エンコーディング:** UTF-8 推奨

---

## 1. 概要

`dirp` はディレクトリ構造の生成専用 DSL です。  
最小構文・予測可能性・厳格なエラー処理を重視します。  
`dirp` は**ファイルを作成せず**、ディレクトリのみを作成します。

## 2. 基本構文

### 2.1 エンティティ
予約文字以外の文字列はディレクトリ名として解釈されます。

### 2.2 兄弟要素の区切り
同一階層は次のいずれかで区切ります（同等扱い）:
- `,`
- `|`
- 改行（`\n`）

### 2.3 親子階層
`{ ... }` で子階層を表現します。
- `A { B }` は、`A/B` を意味します
- 入れ子の深さに理論上制限はありません

### 2.4 空白
- 名前の前後の空白はトリムされます
- 名前の内部空白は保持されます

---

## 3. テンプレート関数

1つのエンティティ名に含められるテンプレート関数は**最大1つ**です。  
関数の多重利用・再帰利用は無効です。

### 3.1 範囲関数 `#(start,end,step)`
整数範囲を生成します。
- 例: `node_#(1,5,2)` -> `node_1`, `node_3`, `node_5`

### 3.2 リスト関数 `@(item1,item2,...)`
列挙値を生成します。
- 例: `srv_@(web,db)` -> `srv_web`, `srv_db`

### 3.3 制約
- 括弧内の区切りは `,` のみ
- 引数間の改行は許容（値/数値の途中改行は不可）
- `name_#(1,3)_@(a,b)` のような複数テンプレートは無効

---

## 4. エラー処理（Fail-Fast）

以下は即時エラー:
1. 空名（例: `A,,B`, `{,}`）
2. 予約文字の不正使用（`{ } ( ) , | # @`）
3. 括弧/波括弧の不整合
4. テンプレート引数の数・型不正
5. テンプレートの再帰/入れ子利用

---

## 5. システム動作

### 5.1 既存ディレクトリ
対象ディレクトリが既存の場合はスキップし、処理を継続します。

### 5.2 文字と互換性
- UTF-8 推奨
- 非ASCII文字可否はホストOS/ファイルシステム依存
- `#`, `@` は ASCII 半角であること

### 5.3 1行実行
本DSLは1行に圧縮しても同じ意味を保つ設計です。

---

## 6. 拡張方針

正規表現や複雑な動的計算は `dirp` 本体に入れず、  
親スクリプト言語側で文字列を組み立てて `dirp` に渡す方針とします。
